Визуальная проверка качества UI.

Цель: $ARGUMENTS

## Режимы работы:

### Если указан путь к скриншоту:
1. Прочитай изображение по указанному пути
2. Проанализируй:
   - Выравнивание элементов
   - Консистентность отступов
   - Читаемость текста
   - Контрастность цветов
   - Адаптивность (если виден viewport)
   - Соответствие дизайн-токенам проекта

### Если указан компонент/экран:
1. Найди файл компонента в проекте
2. Проанализируй код на:
   - Использование Tamagui-токенов (не хардкод цветов/размеров)
   - Правильные отступы ($space.*)
   - Адаптивность (useMedia или responsive props)
   - Accessibility (accessibilityLabel, accessibilityRole)
   - Тёмная тема (поддержка $color/$background)
3. Проверь по чеклисту:
   - [ ] Токены вместо хардкода
   - [ ] Тёмная/светлая тема
   - [ ] Мобильный viewport (375px)
   - [ ] Accessibility labels
   - [ ] Анимации из packages/ui/src/animations/
   - [ ] Нет overflow на мобильном
