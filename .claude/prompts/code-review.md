# Code Review Checklist — MVPTemplate

Проведи ревью изменённого кода по следующему чеклисту.

## 1. Архитектура
- [ ] Код в правильном пакете/модуле (не смешаны зоны ответственности)
- [ ] Backend: паттерн routes → service → repository соблюдён
- [ ] Frontend: компоненты в packages/ui, экраны в apps/mobile/app/
- [ ] Нет циклических зависимостей между пакетами
- [ ] Переиспользуются существующие утилиты и компоненты

## 2. Кросс-платформа
- [ ] Нет `<Modal>` на web — используется AppModal
- [ ] Platform.OS проверки где необходимо
- [ ] MMKV lazy require для native, localStorage для web
- [ ] Нет web-only API без проверки (window, document)
- [ ] Tamagui токены вместо хардкод значений

## 3. Безопасность
- [ ] Нет SQL-инъекций (Drizzle parametrized queries)
- [ ] Нет XSS (sanitization пользовательского ввода)
- [ ] Auth middleware на защищённых маршрутах
- [ ] Нет секретов в коде (API keys, passwords)
- [ ] Zod-валидация на всех входных данных API

## 4. Производительность
- [ ] Нет лишних ре-рендеров (React.memo, useMemo, useCallback где нужно)
- [ ] Списки используют FlatList/VirtualizedList, не map()
- [ ] Тяжёлые вычисления вынесены
- [ ] Изображения с правильными размерами
- [ ] Нет N+1 queries в backend

## 5. i18n
- [ ] Все пользовательские строки через useAppTranslation()
- [ ] Ключи добавлены во все 4 локали (en, ru, es, ja)
- [ ] Нет хардкод текстов в JSX
- [ ] Плюрализация учтена

## 6. Тестирование
- [ ] Unit-тесты для новой бизнес-логики
- [ ] Edge cases покрыты
- [ ] Моки корректны (не тестируем моки)
- [ ] Тесты изолированы (не зависят от порядка)

## 7. Качество кода
- [ ] TypeScript strict — нет any, as, @ts-ignore без причины
- [ ] Именование понятное и консистентное
- [ ] Нет дублирования кода
- [ ] Комментарии только где логика неочевидна
- [ ] Нет console.log (кроме отладки с TODO)

## Формат вывода:
```
=== CODE REVIEW ===

Оценка: A/B/C/D
Критичных: N | Важных: M | Мелких: K

Критичные:
1. [файл:строка] Описание проблемы

Важные:
1. [файл:строка] Описание

Рекомендации:
1. Описание улучшения
```
